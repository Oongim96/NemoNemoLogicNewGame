# 06. 잉크 덱 빌드 (컨셉 능력)

[← 구역·칸 보상](./05-sections-and-rewards.md) · [목차](./README.md) · [다음: 자동 전투 →](./07-auto-battle.md)

퍼즐을 **그냥 푸는 것보다 재밌게** 만들고, 마지막 **자동 전투 뽕맛**까지 한 줄로 이어지는 핵심 시스템.

런 동안 모으는 **컨셉 능력 카드(잉크 카드)** 를 **덱**으로 쌓는다.  
같은 덱이 **퍼즐 중 패시브·발동 효과**와 **자동 전투 중 자동 스킬** 양쪽에 쓰인다.

---

## 1. 칸 정답 → 보상 (즉시)

정답 칸을 맞출 때마다 그 칸에 배치된 보상이 **즉시** 연다. 구역 완료를 기다리지 않는다.

### 칸 보상 가중치 (예시)

| 보상 | 빈도 | 내용 |
| --- | --- | --- |
| **능력 3택1** | **~45%** | 잉크 카드 초안 3장 중 1장 덱에 추가 |
| 골드 | ~20% | 상점·덱 정리 재화 |
| 상점 칸 | ~12% | 즉시 상점 UI (덱 구매·삭제·변환) |
| 이벤트 칸 | ~10% | 선택지 (교환·합성·저주 등) |
| 회복 | ~5% | HP·스킬 충전 |
| 함정 | ~5% | 디버프, 잘못된 카드 등 |
| 보스 단서 | ~3% | 마지막 전투 약점·연계 힌트 |

**능력 3택1**이 가장 자주 나온다. 로그라이크 "한 판 빌드 쌓기"의 주된 재료.

```txt
[칸 정답!]  ■
  → 「잉크 카드 선택」 3택1
     [연쇄 채색] [붉은 잉크] [줄 완성의 메아리]
  → 1장 선택 → 덱에 추가 → 퍼즐 계속
```

---

## 2. 잉크 카드 = 컨셉 능력

한 장의 카드에 **퍼즐 효과** + **전투 효과** + **컨셉(태그)** 가 함께 있다.

### 카드 구성

| 요소 | 설명 |
| --- | --- |
| 이름 | 예: 「연쇄 채색」「붉은 잉크」「줄 완성의 메아리」 |
| 컨셉 태그 | `잉크` `불꽃` `달빛` `행운` `철벽` `힌트` … — 시너지·연계용 |
| 퍼즐 효과 | 패시브 또는 조건 발동 (칸·줄·구역 단위) |
| 전투 효과 | 합산 공격·버프·스택 (쿨타임 있음) |
| **전투 쿨** | 1~4턴 — 발동 후 N턴 재사용 불가 |
| 등급 | 일반 / 희귀 / 에픽 — 효과량·태그 수 |

### 퍼즐 쪽 발동 타이밍

| 타이밍 | 예시 |
| --- | --- |
| **패시브** | 항상 힌트 1줄 강조, 실수 피해 -1 |
| **칸 정답** | 인접 칸 1개 상태 공개 |
| **줄/열 완성** | 해당 줄 숫자 힌트 재검증, 골드 +5 |
| **구역 완료** | 다음 구역 시작 시 랜덤 칸 2개 공개 |
| **액티브 (N회)** | 3x3 영역 정답 표시 — 충전식 |

퍼즐 중 카드가 발동할 때마다 **짧은 이펙트 + 사운드**로 "덱이 돕고 있다"는 체감.

### 전투 쪽 (자동 · 턴 합산)

자동 전투는 **턴제**. 매 턴 **쿨 0인 카드를 전부** 모아 **한 번에 합산·중첩** 공격 → 적 **1회 반격**.  
상세: [자동 전투](./07-auto-battle.md)

- 카드마다 **턴 쿨** (1~4). 쓴 턴 다음엔 빠짐 → 턴마다 조합이 바뀜
- 공격·버프 **동시 발동 후 합산** (뽕맛)
- 버프 카드도 쿨 있음 — 다음 턴엔 안 쓰이지만 지속 효과는 별도 가능
- 파티 ult: 해당 턴 합산에 가산

```txt
T1: [연쇄×2 + 붉은잉크] 합산 520 → 적 HP 감소 → 적 1타
T2: [연쇄×2만] (붉은잉크 쿨2) 합산 160 → 적 1타
T3: [연쇄×2 + 붉은잉크] 합산 680 → …
```

## 3. 덱 빌딩 규칙

| 규칙 | 값 (예시) |
| --- | --- |
| 덱上限 | 12 ~ 18장 (난이도·영구 업그레이드로 확장) |
| 시작 덱 | 파티 캐릭터별 1~2장 + 공통 2장 |
| 중복 | 같은 카드 최대 2장 (3장째는 자동 변환 or 거부) |
| 덱 초과 | 3택1 시 덱 full → **교체**(기존 1장 버리고 새 카드) or **업그레이드** 제안 |

### 빌드 방향 예시

- **잉크 폭발**: `잉크` 태그만 모아 스택 → 한 턴에 터뜨리기
- **줄 완성 러시**: 퍼즐에서 줄 완성 → 전투에서 연속 발동
- **힌트 탐험**: 퍼즐 정보 advantage → 실수 적음 → 전투 버프 유지
- **저주 하이리스크**: 함정·저주 카드 섞고 전투 위력 극대화

---

## 4. 상점 (덱 편집 허브)

**상점 칸** 정답 or **구역 완료 보너스** or **골드로 호출** 가능.

| 메뉴 | 설명 | 비용 |
| --- | --- | --- |
| **카드 구매** | 진열 4~6장 중 1장 덱 추가 | 골드 |
| **카드 삭제** | 덱에서 1장 영구 제거 (덱 thinning) | 골드 |
| **카드 변환** | 1장 → 같은 등급±1 랜덤 카드로 교체 | 골드 + 재료 |
| **카드 강화** | +1 강화 (퍼즐·전투 수치 소폭 UP) | 골드 + 조각 |
| **덱 리롤** | 다음 3택1 후보 풀 1회 재굴림 | 골드 |
| **슬롯 확장** | 덱上限 +1 (1런 1~2회) | 골드 (비쌈) |

상점은 **전투력 포션**보다 **덱 편집** 비중이 크다.  
"지금 덱이 너무 클다 → 삭제", "방향 안 맞다 → 변환", "키 카드 찾았다 → 강화".

---

## 5. 이벤트 (덱 변형)

| 이벤트 | 선택지 예 |
| --- | --- |
| 잉크의 거울 | HP 10% → 에픽 카드 1장 / 거절 |
| 두 카드의 합 | 덱 카드 2장 → 1장 희귀+로 **합성** |
| 저주 각인 | 카드 1장에 저주(전투 -HP) + 전투 위력 2배 |
| 잃어버린 페이지 | 카드 1장 삭제 → 골드 2배 |
| 모르는 상인 | 상점 50% 할인 vs 다음 함정 칸 +1 |
| 운명 재작성 | 덱 전체 리롤 vs 현재 덱 유지 + 골드 |

이벤트도 **덱 모양을 바꾸는** 쪽이 많다.

## 6. 카드 예시

### 연쇄 채색 (일반 · 잉크) — **전투 쿨 1**

- **퍼즐:** 칸 정답 시, 십자 인접 칸 중 오답 1개 X 처리
- **전투:** 80 피해 (합산) + 잉크 스택 +1

### 붉은 잉크 (희귀 · 잉크·불꽃) — **전투 쿨 2**

- **퍼즐:** 실수 1회당 공격력 스택 +1 (런 내 유지)
- **전투:** 잉크 스택 전부 소모 → 소모량 × 80 피해 + 공격력 +15% (해당 턴)

### 줄 완성의 메아리 (일반 · 달빛) — **전투 쿨 3**

- **퍼즐:** 줄/열 완성 시 해당 줄 힌트 1초 강조
- **전투:** 줄 완성 누적 × 15 피해 (합산)

### 황금 자 (일반 · 행운) — **전투 쿨 2**

- **퍼즐:** 줄 완성 시 골드 +3
- **전투:** 해당 턴 합산 피해 +10% (버프, 중첩 가능)

### 무한 지우개 (희귀 · 힌트) — **전투 쿨 3**

- **퍼즐:** 구역당 오답 1회 무효
- **전투:** 파티 실드 40 (합산, 해당 턴)

### 뒤집힌 지도 (에픽 · 미스터리) — **전투 쿨 4**

- **퍼즐:** 일부 구역 타입 숨김, 대신 3택1 등급 +1
- **전투:** 숨겨진 구역 1개당 전투 시작 버프 +5%

## 7. 컨셉·합산 규칙 (요약)

카드마다 다르지만 **타입별 기본**은 고정:

| 타입 | 겹칠 때 |
| --- | --- |
| 버프 | 수치 **합산** |
| 공격 | **타격 횟수** +1 (불꽃 등 연타 컨셉) |
| 실드 | 실드량 **합산** |
| 스택 | 자원 **합산** |

**덱에 N장** / **이번 턴 N장 발동** → 컨셉마다 **공격 방식이 바뀜** (범위화, 연타 화상, 만월 크리, 반격…).

→ 상세 8컨셉·임계치·교차 시너지: **[컨셉 카탈로그](./06b-concept-catalog.md)**

### 태그 시너지 (예시)

| 조건 | 효과 |
| --- | --- |
| `잉크` 턴 3장+ | 잉크 범위 공격 |
| `불꽃` 턴 연타 5+ | 마지막 타격 ×2 |
| `달빛`+`힌트` | 약점 2배 |
| `격자` 덱 5장+ | 줄 완성 → 전투 배율 누적 |

파티 캐릭터는 특정 컨셉 **덱·턴 임계 +1** or **교차 보너스** (09 참고).

## 8. 퍼즐 재미를 위한 설계 원칙

1. **칸마다 뭔가 남는다** — 3택1, 골드, 상점, 이벤트 중 하나는 거의 항상
2. **덱이 퍼즐을 바꾼다** — 힌트·줄 완성·실수 완화로 "그냥 네모네모"가 아님
3. **선택의 누적** — 3택1 + 상점 삭제/변환으로 **내 빌드**가 보임
4. **전투는 덱 쇼케이스** — 같은 카드가 퍼즐에서 이미 도와줬으니 전투에서 터질 때 납득
5. **구현 일관성** — 카드 데이터 1벌, 퍼즐 트리거 / 전투 트리거만 분기

## 9. 구현 스케치 (참고)

```txt
InkCard { id, name, tags[], grade, battleType: attack|buff|shield|stack|special,
          puzzle: { trigger, effect },
          battle: { cooldown, baseValue, mergeRule } }

// 턴 합산
resolveTurn(cards) {
  buffs = sumSame(buffCards)
  hits = count(attackCards) + conceptThresholdBonus()
  shields = sum(shieldCards)
  applyConceptTransforms(tags, deckCount, turnCount)  // 잉크범위, 불길통로…
  damage = calc(hits, buffs, stacks)
}
```
